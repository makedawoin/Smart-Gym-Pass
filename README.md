We built a complete gym management system called Smart Gym Pass System where members can join, buy passes, check in, and track their gym visits. The system keeps track of everything - members, passes, check ins, and payments.
We're a team of 5 people: Feven built the database connection so the program can talk to MySQL and also handles all payment saving and tracking, Hasset created the pass system with 3 types of passes (Trial, Monthly, Yearly) where each pass type has different prices and durations, Huda built the attendance tracking system that records when members check in and check out, Makeda created the member profiles and member management handling everything about gym members, and Meron built the exception handling system with custom exceptions and the reporting system so admins can see reports. When you run the program, you can add new members to the gym, view member details, update member information, issue passes to members (Trial, Monthly, or Yearly), record when members check in and check out, process payments, and view detailed reports about members, passes, check ins, and money. We used Java as the programming language, MySQL as the database where we store all the data, and JDBC as the bridge between Java and MySQL. To set up, download and install MySQL Server on your computer (you can leave password empty or set one), the program creates the database automatically when you run it so no setup needed, in IntelliJ go to Project Structure and add the MySQL JDBC driver, if you set a password when installing MySQL open DatabaseConnection.java and put your password where it says private static final String DB_PASSWORD = "", then right click SmartGymPassSystem.java and click Run. When you open the program you see a menu with 6 options: Member Operations to add/view/update members, Pass Operations to issue or renew passes, Check in/Attendance to record gym visits, Payment Operations to process payments, Reports & Analytics to view reports, and Exit to close the program. Our file structure has core classes DatabaseConnection.java and SmartGymPassSystem.java, member classes Member.java and MemberManager.java, pass classes Pass.java, MonthlyPass.java, YearlyPass.java, TrialPass.java, and PassManager.java, check-in classes CheckIn.java and AttendanceTracker.java, payment classes Payment.java and PaymentProcessor.java, exception classes InvalidPassException.java and ExpiredPassException.java, and report class GymReport.java. When you run the program for the first time we automatically create 2 test members: Abebe (M001) with a monthly pass and Almaz (M002) with a yearly pass so you can test the system or add your own members. We learned all 7 chapters of OOP: Encapsulation with private fields and getters/setters, Inheritance where Pass classes extend the abstract Pass class, Polymorphism where different pass types calculate renewal cost differently, Abstraction with abstract Pass class and abstract methods, Exception Handling with custom exceptions for errors, Files & Streams with reports ready to save, and JDBC for database operations. If you get errors like "Cannot resolve symbol Pass" make sure all class files are in the same folder and check that Pass classes are created, if you get "Connection refused" make sure MySQL Server is running and check that password matches in DatabaseConnection.java, and if you see "Table already exists" this is normal because the program checks if tables exist before creating them so just ignore it. We could add more features in the future like member login system, email notifications, membership renewal reminders, trainer scheduling, and workout history tracking. If something doesn't work check that MySQL is running, all Java files are in the same project, JDBC driver is added to project, and password is correct in DatabaseConnection.
